以下 5 个硬编码用户确认在当前使用场景（认知科学、心理学、计算机、社科类书籍，逐本编译）中合理：

### 仍需关注的中等硬编码

**[中等] 1. `tests/common/mod.rs:369-389` — mock_load_prompt 硬编码 prompt 模板**

所有 14 种 prompt 模板完全硬编码在测试代码里。当真实 `prompts/` 目录下的文件更新后，测试将使用过时的模板。Stage 缓存 key 基于 prompt hash，会导致测试验证的行为与实际行为不一致。

**修复建议**: 测试应从实际 `prompts/` 目录加载，或设置 `CARDNOTE_PROMPTS_DIR` 环境变量。

**[中等] 2. `src/quality/card_lint.rs:52-59` — CardLintConfig 默认值**

`min_content_length: 30`、`max_garbage_ratio: 0.3`、`min_info_density: 0.05` 对用户手写的短卡片（如诗歌金句）可能过严。

**[中等] 3. `src/config.rs:679-691` — load_books_config 硬编码"阳志平"默认书籍**

默认配置包含特定作者的书籍列表。不是该读者的用户完全不需要此默认值。

**修复建议**: `.cardnote/books.json` 不存在时返回空列表。

## 二、重复代码

### [严重] 4. CardType 名称映射重复三次

| 位置 | 函数 | 映射内容 |
|------|------|---------|
| `src/models.rs:27-43` | `CardType::as_str()` | CardType → 中文名称 |
| `src/stages/cards.rs:139-153` | `CardPlanner::card_type_name()` | CardType → 中文名称（完全相同） |
| `src/stages/cards.rs:252-267` | `card_type_prompt_name()` | CardType → prompt 文件名 |

`as_str()` 和 `card_type_name()` 是完全重复的映射。`card_type_prompt_name()` 本质相同（英文名 → snake_case）。

**修复建议**: `card_type_name()` 委托给 `CardType::as_str()`。

### [严重] 5. compute_info_density 两个独立实现不一致

| 位置 | 实现方式 |
|------|---------|
| `src/dedup.rs:314-332` | 硬编码 5 个关键词（"「」"、"【】"、"比如"、"例如"、"关键"、"核心"） |
| `src/quality/card_lint.rs:464-504` | 可配置的 density_markers + 权重系统 |

两个同名函数计算"信息密度"，但算法完全不同。dedup 中的简单版修改后不会影响 card_lint，导致评分体系内部不一致。

**修复建议**: dedup.rs 中的 `compute_info_density` 应调用 card_lint 中的版本。

### [严重] 6. 去重和实体统一的"相似度图→连通分量→合并"逻辑重复

| 位置 | 用途 |
|------|------|
| `src/dedup.rs:196-245` | 卡片语义去重 |
| `src/stages/entities.rs:158-205` | 实体统一 |

两者都做 O(n²) 全对比较 → 相似度判断 → 构建邻接图 → DFS 找连通分量 → 组内 canonical 合并。差异仅在相似度判断函数（Jaccard vs 规范化名称匹配）。

**修复建议**: 抽取通用的 `group_by_similarity` 抽象。

### [中等] 7. 标题正则 `^(#{1,3})\s+` 在 4 个文件中重复编译

| 文件 | 用途 |
|------|------|

| `src/quality/metrics.rs:98` | 结构完整性评分 |
| `src/quality/preprocess.rs:225` | 换行合并 |

**修复建议**: 提升 `pipeline.rs` 中的 `heading_regex()` 为 `pub` 函数到 `lib.rs`。
