## 二、P0 问题：ref 格式违规

### 2.1 问题描述

54 张卡片中，ref 字段存在 **4 种格式变体**，严重违反 prompt 规范。

Prompt 规范：
> 格式：`来源名_p数字`
> 规则：必须引用当前正在阅读的文档，格式必须严格为 `来源名_p数字`
> 反例：`文档第236页` — "文档"不是具体名称

### 2.2 违规详情

#### 类型 A："本书"替代书名（6 处）

| 行号 | ref 内容 | 对应卡片 |
|------|---------|---------|
| 299 | `本书_p172` | 写作拖延的根源 |
| 318 | `本书_p172-173` | 写作的认知单位不是词汇 |
| 736 | `本书_p203-204` | 有趣不是天性使然 |
| 774 | `本书_p202-203` | 有趣是人格与社会性的偏离 |
| 1002 | `本书_p215-217` | 读书通透的秘密 |
| 1021 | `本书_p210-211` | 风格的秘密是时空变形 |

**问题**："本书"不是具体名称。一年后回看，无法追溯来源。

#### 类型 B：章节名替代书名（4 处）

| 行号 | ref 内容 | 对应卡片 |
|------|---------|---------|
| 394 | `行动模式_p111-p112` | 刻意练习的关键是攒硬盘 |
| 413 | `行动模式_p120-p123` | 动机的真正源泉 |
| 641 | `行动模式_p116-p119` | 新年计划失败的原因 |
| 717 | `行动模式_p113-p115` | 最佳成人学习方式 |

**问题**："行动模式"是书中篇章名，不是书名。格式不统一。

#### 类型 C：多余书名号（4 处）

| 行号 | ref 内容 | 对应卡片 |
|------|---------|---------|
| 14 | `《人生模式》_p272-p276` | 数字土著 |
| 128 | `《人生模式》_p269-p271` | 人类社会由信息浓度决定 |
| 204 | `《人生模式》_p270-p272` | 人生生态位决定职业天花板 |
| 698 | `《人生模式》_p277-p280` | 暴力的本质是熵增 |

**问题**：prompt 要求格式为 `来源名_p数字`，不含书名号。

#### 类型 D：前导零不统一（1 处）

| 行号 | ref 内容 | 对应卡片 |
|------|---------|---------|
| 109 | `人生模式_p049` | 人生困惑的价值 |

**问题**：其他卡片均无前导零（如 `p41`、`p86`），此处 `p049` 格式不一致。

### 2.3 用户确认

用户确认：连字符范围（如 `p220-221`）是允许的。其余格式违规均需修复。

### 2.4 修复方案

**方案一：Prompt 层加强约束（推荐）**

在 `prompts/all_cards.md` 中增加 ref 格式的负面示例：

```markdown
反例（不允许）：
- `《人生模式》_p160` — 多余书名号
- `本书_p172` — "本书"不是具体名称
- `行动模式_p111` — 章节名不能替代书名
- `p049` — 页码不需要前导零
```

**方案二：后处理层格式标准化（兜底）**

在 `output.rs` 或新增 `quality/ref_format.rs` 中，对产出卡片的 ref 字段做正则校验：

```rust
// 合法格式：书名_p数字 或 书名_p数字-数字
// 不合法：带书名号、含"本书"、含章节名、前导零
```

### 2.5 验收标准

- [ ] 全部卡片 ref 格式统一为 `书名_p数字` 或 `书名_p数字-数字`
- [ ] 无书名号、无"本书"、无章节名替代、无前导零
- [ ] ref 格式不合法的卡片标记为 `status: Rejected`

## 三、P0 问题：LLM 编造例子/案例

### 3.1 问题描述

Prompt 明确要求：
> 每张卡片必须能追溯到文档中的具体段落或例子，杜绝凭空编造

但对抗性验证发现，多张卡片的"例子"部分**并非书中的具体案例**，而是 LLM 基于"新知"自行创作的类比或叙事。

### 3.2 编造案例详情

#### 案例 1：数字土著卡片

**ref**：p272-p276

**卡片中的"例子"**：
> 一个"90后"年轻人，可以在一个晚上认识一个跨多个行业的虚拟社群里的几十个核心成员（150-500人），并通过社交媒体与他们保持高频互动，形成弱关系网。而一个"70后"管理者，可能花了十年才在同一个公司内部建立了50个强关系。

**PDF 原文验证**（p272-p276）：原文讨论的是邓巴数字、信息-信任生态位、数字土著的认知跃迁。**无此具体对比案例**。

**判断**：🔴 LLM 编造

---

#### 案例 2：不吵架的夫妻卡片

**ref**：p259-261

**卡片中的"例子"**：
> 走在街上，看到一对夫妻，妻子因为丈夫忘记买菜而严厉地数落他，丈夫也据理力争，最后两人气呼呼地牵手去超市。

**PDF 原文验证**（p259-261）：原文讨论的是中国婚恋文化中的"外人"与"自己人"阶段、吵架作为关系认证仪式。**无此具体街头场景**。

**判断**：🔴 LLM 编造

---

#### 案例 3：转型路上最大障碍卡片

**ref**：p134

**卡片中的"例子"**：
> 一个在国企做了10年财务的人，想转行做户外探险领队。他告诉妻子，妻子会说："你那么沉静，怎么能做那个？"

**PDF 原文验证**（p134）：原文讨论的是职业转型期的"旧身份茧房"。**无此具体人物故事**。

**判断**：🔴 LLM 编造

---

#### 案例 4：小县城银行的小王卡片

**ref**：p269-p271

**卡片中的"例子"**：
> 小王在小县城银行工作（低信息-低信任生态位），生活安逸。他辞职加入一个高速发展的 AI 创业公司...

**PDF 原文验证**（p269-p271）：原文讨论的是信息-信任四象限、生态位跃迁。**无"小王"此具体人物**。

**判断**：🔴 LLM 编造

---

#### 案例 5：人格矛盾/有趣系列（多张卡片）

"人格矛盾是有趣的前提"（p250）卡片中的例子：
> 想象一位性格内向的作家，一边婉拒热闹的酒局，一边主动邀请三五知己深谈至深夜。

**PDF 原文验证**（p250）：原文引用的是**一位读者的真实来信**（"一位读者来信说...从动机上来说，她是一个高合群的人..."）。

**问题**：LLM **将书中的真实案例（读者来信）替换成了完全虚构的作家故事**。不仅编造，还**篡改了证据来源**。

**判断**：🔴 LLM 编造 + 篡改证据

---

### 3.3 根因分析

LLM 为什么会编造例子？

1. **Prompt 激励偏差**：prompt 要求"用自己的话写"，LLM 将"例子"也视为需要"原创"的部分
2. **信息密度压力**：prompt 要求 200-500 字/卡，LLM 认为编造一个"生动的"例子比复述书中案例更能满足字数要求
3. **缺乏负向约束**：prompt 中虽然写了"杜绝凭空编造"，但没有在"例子"部分给出具体的负面示例

### 3.4 修复方案

**方案：Prompt 层增加"例子"专项约束**

在 `prompts/all_cards.md` 中，对"例子"部分增加明确规则：

```markdown
## 例子字段规范

1. **例子必须来自原文** — 优先使用书中提到的具体研究、真实人物故事、作者亲历案例
2. **禁止 LLM 自行创作类比故事** — 不允许使用"想象一下...""比如有一个人..."等虚构叙事
3. **如果原文无具体例子** — 可以省略"例子"字段，或用一句话概括原文的论证逻辑，绝不编造
4. **负向示例**：
   - ❌ "想象一位性格内向的作家..." — LLM 自行创作的虚构人物
   - ✅ "书中引用了一位读者的来信..." — 来自原文的具体案例
```

### 3.5 验收标准

- [ ] 全部卡片的"例子"部分可追溯至 PDF 原文的具体段落
- [ ] 无"想象一下""比如有一个人""假设你..."等 LLM 虚构叙事开头
- [ ] 无法验证的例子标记为 `status: NeedsRetry`，reason: "例子无法追溯"

## 四、P1 问题：主题重复应合并

### 4.1 问题描述

54 张反常识卡中，**"有趣/人格冲突/动机特质"主题出现了 5 张**，核心论点高度重叠。

### 4.2 重复卡片列表

| # | 标题 | ref | 核心论点 |
|---|------|-----|---------|
| 1 | 人格矛盾，不是缺陷，而是"有趣"的前提 | p250 | 人格特质与动机错配 = 有趣 |
| 2 | 你的人格冲突，不是性格缺陷，而是高级趣味的来源 | p248-p250 | 同上，换角度阐述 |
| 3 | 有趣的人不是天性使然，而是人格发展与社会性发展的偏离博弈 | p202-203 | 有趣 = 长期偏离社会规范 |
| 4 | 有趣不是幽默，而是"不装" | p244-p247 | 有趣 = 真诚放松 |
| 5 | 有趣不是"个性"，而是长期偏离社会规范的稳定风格 | p203-204 | 有趣 = 长期偏离形成的风格 |

**问题**：这 5 张卡片如果同时出现在 Anki 复习中，用户会在短时间内反复看到"人格冲突/偏离规范 = 有趣"的变体。这不是"一卡一事实"，而是**一主题多卡片**。

### 4.3 用户确认

用户确认：主题重复应合并为一张。

### 4.4 修复方案

**方案：后处理层增加语义去重（轻量版）**

在 `dedup.rs` 中，不仅按 `(title, type)` 去重，还增加**主题相似度检测**：

```rust
// 如果两张反常识卡的核心论点相似度 > 阈值，提示合并
// 检测方法：提取"新知"部分的关键词重叠率
```

**合并策略**：
- 保留信息最完整的一张作为"主卡"
- 将其他重复卡片的内容作为"补充视角"并入主卡
- 或标记为 `status: Merged`，在质量报告中标注

### 4.5 验收标准

- [ ] 同一主题（核心论点重叠 > 70%）的卡片合并为一张
- [ ] 合并后的卡片保留最完整的 ref 和例子
- [ ] 被合并的卡片在质量报告中标注合并原因

### 5.1 问题描述

部分卡片依赖书中首次出现的概念，但卡片内未给出足够解释，导致脱离原书后难以理解。

### 5.2 问题卡片列表

| 卡片标题 | 依赖概念 | 问题 |
|---------|---------|------|
| 焦虑不是敌人 | STC 算子 | 卡片内未解释什么是 STC 算子 |
| 人际关系的本质是自我边界 | 差序格局 | 首次出现，未解释费孝通的理论 |
| 信息爆炸的第一位牺牲品 | 邓巴数字、意向层次 | 两个专业术语均未解释 |
| 先行动再思考 | 可能的自我 | 未解释伊瓦拉的理论背景 |

### 5.3 修复方案

**方案：卡片内术语首次出现时必须附带简要解释**

在 `prompts/all_cards.md` 中增加：

```markdown
## 术语处理规范

1. 如果卡片中使用了书中首次出现的专业术语，必须在同一卡片内给出 1-2 句话的解释
2. 解释应放在"新知"部分，用括号或独立句子呈现
3. 不要假设读者已经知道这个概念

示例：
- ❌ "用 STC 算子分析..." — 读者不知道 STC 是什么
- ✅ "用 STC 算子（空间-时间-成本三维分析工具）分析..." — 自带解释
```

### 5.4 验收标准

- [ ] 每张卡片中使用的专业术语在同一卡片内有简要解释
- [ ] 解释长度控制在 20 字以内，不喧宾夺主

## 附录 A：术语卡专项审计（99 张）

> 审计说明：用户已确认以下两项为设计意图，不作为问题：
> - `loc` 字段：标注术语在本书中出现的位置（与 `ref` 原始出处区分）
> - 术语卡标题问句化：如"什么是 MCII 方法？"——用户认为可以接受
> - 同一术语多张卡片：等同于"主题重复"（见第 4 节），不再重复记录

### A.1 P0：ref 来源变成了其他书名

4 张术语卡的 ref 直接写成了**另一本书的书名**，而非《人生模式》：

| ref | 卡片 | 问题 |
|-----|------|------|
| `人类的演化_p333` | 社会脑 | 来源变成了另一本书 |
| `改变_p327` | 第二序改变 | 来源变成了另一本书 |
| `风格感觉_p331` | 古典风格 | 来源变成了另一本书 |
| `行动模式_p...`（4 处） | 刻意练习等 | 来源变成了篇章名 |

**修复方案**：如果术语原始出处是另一本书，ref 应写为 `人生模式_p...（引用自《XXX》）`，确保来源可追溯至当前阅读文档。

### A.2 P0：质量门控失效

第 1505-1507 行：

```
**质量：** 70% / Accepted
**处理原因：** 标题与内容不匹配
```

系统已检测出"自我决定论"卡片存在**标题与内容不匹配**的质量问题，但仍标记为 `Accepted` 并输出。

**根因**：质量过滤逻辑 `filter_cards_with_source` 可能将 `quality_score < 1.0` 的阈值设得过低，或 `status` 字段的降级逻辑未生效。

**修复方案**：
- 将 `status != Accepted` 或 `reject_reason` 非空的卡片标记为 `NeedsRetry` 或 `Rejected`
- 在 `output.rs` 输出前增加拦截：任何带 `reject_reason` 的卡片不得进入最终输出

### A.3 P0：类型混淆

术语卡文件中出现了一张**标题为"反常识卡"的卡片**（第 492 行）：

```
**标题：** 反常识卡（注：虽任务要求出一张术语卡...）
```

这张卡片讨论的是"反常识卡"这个**卡片类型本身**，而非《人生模式》书中的术语。这是类型错误。

**修复方案**：在卡片生成阶段增加类型校验——术语卡的标题必须是书中的术语/概念名称，不能是其他卡片类型的元描述。

### A.4 P1：术语卡解释过度膨胀

prompt 限制 200-500 字/卡，但术语卡的**解释部分**经常单独超过 300 字：

| 卡片 | 解释字数估算 |
|------|------------|
| 元认知 | ~350 字 |
| 自我决定论 | ~400 字 |
| 社会比较 | ~450 字 |
| 信息信任四象限 | ~350 字 |

术语卡的"解释"应**简明扼要**（100-200 字），让用户快速理解核心含义。当前 LLM 把术语卡写成了微型百科条目。

**修复方案**：在 `prompts/term_card.md` 中增加解释字数约束：

```markdown
解释长度限制：100-200 字。只解释核心机制，不展开历史背景、应用场景、细分子概念。
```

**根因分析**：《人生模式》书中可能引用了或推荐了这些书（如附录书单），LLM 在生成综述卡时混淆了"书中提到的书"和"当前阅读的文档"，把书中推荐的其他书当成了卡片来源。

**与其他类型的差异**：反常识卡、术语卡、新知卡中**没有出现**此问题（它们最多是 ref 格式问题，但来源还是《人生模式》）。这是综述卡特有的问题。

**修复方案**：
- 在 `prompts/review_card.md` 中增加明确约束："综述卡的 ref 必须引用当前正在阅读的文档《人生模式》，不可引用书中推荐的其他书籍"
- 如果综述内容涉及书中推荐的其他书，ref 仍应标注为 `人生模式_p...（书中推荐了《XXX》）`

### B.2 P1：ref 系统性包含作者名 + 书名号

**18 张综述卡中，17 张的 ref 包含 `阳志平《...》` 前缀**：

```
阳志平《人生模式》_p268-281
阳志平《人生标准九》_p126-p141
阳志平《人生模式》_p282-287
...
```

这是**系统性违规**——不是偶发错误，而是综述卡稳定地输出包含作者名和书名号的 ref。

**根因**：综述卡 prompt 可能缺少 ref 格式示例，或 LLM 将"综述"理解为"学术论文引用格式"而自动添加了作者名和书名号。

**修复方案**：在 `prompts/review_card.md` 中增加 ref 格式示例：

```markdown
ref 格式示例：人生模式_p268-281
注意：不要加作者名，不要加书名号
```

### B.3 P1：同一篇章多页码范围重复

**"《人生模式》第四篇 人际模式 综述" — 3 张**：

| # | ref |
|---|-----|
| 1 | p295-305 |
| 2 | p235-250 |
| 3 | p250-268 |

3 张卡片都是同一本书同一篇章的综述，只是覆盖的页码范围不同。

**修复方案**：与主题重复合并处理——同一篇章的综述卡应合并为一张，或根据页码范围判断是否保留最完整的一张。
