# 2026-05-26 卡片示例必含元素与 Prompts 对照分析

## 一、卡片示例中的必含元素总结

已阅读 `02-聪明的阅读者卡片示例` 目录下 21 个 Markdown 示例；以下是按“卡片类型”归纳出的每一种卡片示例的必含元素。

`★ Insight ─────────────────────────────────────`
卡片的“必含元素”可以分两层看：一层是所有卡片共享的元数据，如标题、参考、唯一编码；另一层是卡片类型决定的内容结构，如术语卡必须有定义，事件卡必须有时间/地点/行动者/反应。
`─────────────────────────────────────────────────`

### 1. 所有卡片的通用必含元素

几乎所有示例都包含这几类：

| 元素 | 作用 | 说明 |
|---|---|---|
| 标题 / 主题 | 标识这张卡在讲什么 | 有些写作格式是 `标题：`，有些直接首行写主题 |
| 核心内容 | 卡片主体 | 不同卡片类型的结构不同 |
| 参考 / ref | 溯源 | 书名、文章名、作者、出版信息或课程来源 |
| 唯一编码 / uuid / uid | 唯一标识 | 多为年月日时分格式，如 `202305021641` |
| 卡片类型标签 | 分类 | 如 `#基础卡`、`#新知卡`、`#术语卡` |

### 2. 各类卡片的必含元素

#### 2.1 基础卡

代表文件：

- `基础卡 示例以及细节.md`
- `基础卡-示例 1.md`
- `基础卡-读《找工作》 1.md`
- `基础卡-读《听听那冷雨》.md`

| 必含元素 | 内容功能 |
|---|---|
| 标题 | 说明阅读对象或摘录对象 |
| 原文 / 内容 | 记录原始材料、阅读内容或核心摘录 |
| 收获 / 评论 / 理解 | 写出自己的理解、感受、评论或提炼 |
| 参考 | 标明来源 |
| 唯一编码 | 标识卡片 |
| `#基础卡` | 类型标签 |

判断标准：基础卡的核心是围绕一段材料，记录“我读到了什么 + 我怎么理解”。它不一定要求提出新概念，也不一定要求行动转化，重点是建立最基础的阅读记录。

#### 2.2 新知卡

代表文件：

- `基础卡-读《找工作》.md`
- `新知卡-认知方式.md`
- `新知卡-如何阅读任何一本书？.md`
- `新知卡示例.md`

| 必含元素 | 内容功能 |
|---|---|
| 标题 / 主题 | 标识新知主题 |
| 已知 | 写出自己原本知道什么 |
| 新知 | 写出新增认知、反常识发现或认知升级 |
| 例子 / 应用 / 推演 | 将新知放入具体场景中理解 |
| 参考 | 标明来源 |
| 唯一编码 | 标识卡片 |
| `#新知卡` | 类型标签 |

判断标准：新知卡的核心是从“已知”到“新知”的认知差异。它不是普通摘抄，而是要呈现“我原来怎么想，现在被什么内容更新了”。

#### 2.3 术语卡

代表文件：

- `术语卡-什么是执行意图？.md`
- `术语卡-示例以及细节.md`

| 必含元素 | 内容功能 |
|---|---|
| 术语 / 标题 | 要解释的概念名称 |
| 定义 | 对术语做清晰界定 |
| 解释 | 说明这个术语为什么成立、如何运作 |
| 例子 | 用具体例子帮助理解 |
| 参考 | 标明来源 |
| 唯一编码 | 标识卡片 |
| `#术语卡` | 类型标签 |

判断标准：术语卡的核心是把一个概念讲清楚。最完整的结构是：

```text
术语是什么？
它是什么意思？
它为什么重要？
它在具体情境中怎么用？
来源是什么？
```

#### 2.4 金句卡

代表文件：

- `金句卡-社会比较.md`
- `金句卡-多年以后.md`

| 必含元素 | 内容功能 |
|---|---|
| 标题 | 标识金句主题 |
| 金句 / 原句 | 记录值得保存的句子 |
| 评论 / 仿写 | 对金句进行解释、评论、转化或模仿 |
| 参考 | 标明来源 |
| 唯一编码 | 标识卡片 |
| `#金句卡` | 类型标签 |

判断标准：金句卡的核心是保存一个高价值表达，并对它做再加工。再加工有两种典型方式：评论，说明这句话为什么重要；仿写，迁移句式、节奏或表达结构。

#### 2.5 新词卡

代表文件：

- `新词卡-微醺而饭.md`

| 必含元素 | 内容功能 |
|---|---|
| 新异词汇 | 记录新鲜、有表现力的词语 |
| 原句 | 保留词语出现的原始语境 |
| 造句 | 将词语迁移到自己的表达中 |
| 参考 | 标明来源 |
| 唯一编码 | 标识卡片 |
| `#新词卡` | 类型标签 |

判断标准：新词卡的核心是从他人的语言中提取一个词，再变成自己的表达资源。它必须有“原句”和“造句”，否则只是词语收藏，不是表达训练。

#### 2.6 事件卡

代表文件：

- `事件卡-有点忧伤，有点美.md`
- `事件卡-夏济安初见李彦.md`

| 必含元素 | 内容功能 |
|---|---|
| 标题 | 标识事件 |
| 时间 | 事件发生的时间 |
| 地点 | 事件发生的地点 |
| 行动者 | 谁参与了事件 |
| 行动 / 经过 | 发生了什么 |
| 反应 / 结果 | 事件如何收束 |
| 参考 | 标明来源 |
| 唯一编码 | 标识卡片 |
| `#事件卡` | 类型标签 |

两种事件卡形态：

| 类型 | 特点 |
|---|---|
| 单一事件卡 | 聚焦一个场景，结构为时间、地点、行动者、反应 |
| 多个事件卡 | 用时间线排列多个连续事件 |

判断标准：事件卡的核心是把抽象叙述还原成具体发生过的事情。它必须回答：何时、何地、谁、做了什么、结果/反应是什么。

#### 2.7 图示卡

代表文件：

- `图示卡-人类的社交网络.md`
- `图示卡-人类的九种主流认知方式（结构阅读）.md`

| 必含元素 | 内容功能 |
|---|---|
| 标题 | 标识图示主题 |
| 说明 | 解释图示表达的结构、模型或关系 |
| 图像 / 图表链接 | 图示卡的核心载体 |
| 参考 | 标明来源 |
| 唯一编码 / uuid | 标识卡片 |
| `#图示卡` | 类型标签 |

判断标准：图示卡的核心是用图像表达结构关系。它不是单纯插图，而是要呈现一种关系、层级、模型、分类或流程。

#### 2.8 人物卡

代表文件：

- `人物卡-丹尼尔•丹尼特.md`

| 必含元素 | 内容功能 |
|---|---|
| 标题 / 人名 | 标识人物 |
| 小传 | 记录人物生平、身份、经历 |
| 代表作品 / 研究方向 | 说明人物贡献 |
| 时间线 / 关键节点 | 呈现人物发展脉络 |
| 参考 | 标明来源 |
| 唯一编码 | 标识卡片 |
| `#人物卡` | 类型标签 |

判断标准：人物卡的核心是用最小篇幅建立一个人物的认知档案。它应回答：这个人是谁，他做过什么，他的关键贡献是什么，为什么值得记。

#### 2.9 自由卡

代表文件：

- `自由卡-文无第一.md`

| 必含元素 | 内容功能 |
|---|---|
| 标题 | 标识主题 |
| 自由正文 | 可以是综合性评论、片段组合、观点整理 |
| 参考 | 标明来源 |
| 唯一编码 | 标识卡片 |
| `#自由卡` | 类型标签 |

判断标准：自由卡的核心是不受固定卡型限制，承载综合性、开放性、跨材料的想法。它常用于多张读书卡片的综合、多个来源材料的融合、一个尚未归入具体卡型的思考片段。

#### 2.10 行动卡

代表文件：

- `基础卡-读《找工作》2.md`

虽然文件名是基础卡，但内容标签是 `#行动卡`。

| 必含元素 | 内容功能 |
|---|---|
| 标题 | 标识行动主题 |
| 原理 | 说明行动依据 |
| 行动 | 写出具体可执行动作 |
| 参考 | 标明来源 |
| 唯一编码 | 标识卡片 |
| `#行动卡` | 类型标签 |

判断标准：行动卡的核心是把一个知识点转化为下一步行动。它必须包含“我依据什么原理”和“我要做什么具体行动”。

### 3. 统一模板

如果后续要把这些卡片做成软件模板，可以抽象为：

```text
标题：
核心字段：
解释 / 评论 / 例子 / 行动：
参考：
唯一编码：
#卡片类型
```

不同卡片类型的“核心字段”不同：

| 卡片类型 | 核心字段 |
|---|---|
| 基础卡 | 原文 + 收获 / 评论 |
| 新知卡 | 已知 + 新知 + 例子 |
| 术语卡 | 定义 + 解释 + 例子 |
| 金句卡 | 金句 / 原句 + 评论 / 仿写 |
| 新词卡 | 新异词汇 + 原句 + 造句 |
| 事件卡 | 时间 + 地点 + 行动者 + 行动 + 反应 |
| 图示卡 | 说明 + 图像 |
| 人物卡 | 小传 + 贡献 / 作品 / 时间线 |
| 自由卡 | 自由正文 + 来源说明 |
| 行动卡 | 原理 + 行动 |

结论：这些示例里最稳定的底层结构是“类型标签 + 核心内容结构 + 来源 + 唯一编码”；真正区分卡片类型的，不是标题和编码，而是中间那组“核心字段”。

## 二、Prompts 与示例卡必要元素的对照分析

只读了 `prompts` 目录，没有修改任何 prompt。下面按“示例卡必含元素”对照现有 prompt 的欠缺点。

`★ Insight ─────────────────────────────────────`
卡片 prompt 的关键不是“字段越多越好”，而是字段必须保护该卡型的核心认知动作：新知卡保护“已知→新知”的差异，术语卡保护“定义→解释→例子”，事件卡保护“时间→地点→行动者→反应”。
`─────────────────────────────────────────────────`

### 1. 总体结论

当前 prompts 已经覆盖了大多数卡片类型：

- 基础卡
- 新知卡
- 术语卡
- 金句卡
- 新词卡
- 事件卡
- 图示卡
- 人物卡
- 行动卡
- 索引卡

但和示例卡相比，主要有 5 类不足：

1. 部分卡型字段和示例卡不完全一致。
2. “参考 + 唯一编码 + 类型标签”在单卡 prompt 中较完整，但 `all_cards.md` 缺失唯一编码和类型标签格式。
3. 新知卡缺少“已知”字段，导致不容易体现认知更新。
4. 事件卡缺少“行动者 / 反应”字段，容易变成普通历史摘要。
5. 基础卡被定义得过窄，和示例里的“原文 + 收获 / 评论”不一致。

### 2. 全局层面的欠缺

#### 2.1 `all_cards.md` 的统一格式太弱

文件：`prompts/all_cards.md`

当前格式是：

```md
类型：
标题：
内容：
参考：
```

欠缺：

| 示例卡必含元素 | 当前是否覆盖 | 问题 |
|---|---|---|
| 标题 | 有 | OK |
| 核心字段 | 弱 | 统一写成“内容”，会抹平不同卡型结构 |
| 参考 | 有 | OK |
| 唯一编码 | 无 | 示例卡普遍有 uuid / 唯一编码 |
| 类型标签 | 弱 | 有“类型”，但没有 `#新知卡` 这种标签 |
| 卡型专属结构 | 无 | 新知卡、术语卡、事件卡等都会被压扁 |

影响：`all_cards.md` 会生成“看起来像卡片，但不像示例卡”的结果。尤其是新知卡、术语卡、事件卡、金句卡，会失去类型特征。

建议补强方向：`all_cards.md` 应该要求不同类型使用对应字段，而不是统一塞进 `内容`。至少应补 `唯一编码` 和 `#卡片类型`。

#### 2.2 质量检查 prompt 的完整性标准不全

文件：`prompts/quality_check.md`

当前完整性检查：

```md
人物卡：是否有生平、贡献、代表作品？
术语卡：是否有定义、解释、例子？
新知卡：是否有新知内容、应用场景？
金句卡：是否有原文、出处、解读？
事件卡：是否有时间、地点、经过、意义？
行动卡：是否有具体步骤、预期效果？
```

欠缺：

| 卡型 | 缺少检查 |
|---|---|
| 基础卡 | 没检查“原文 / 内容 + 收获 / 评论” |
| 新词卡 | 没检查“新异词汇 + 原句 + 造句” |
| 图示卡 | 没检查“说明 + 图像 / 图示描述” |
| 自由卡 | 没检查 |
| 索引卡 | 没检查 |
| 所有卡 | 没检查唯一编码、类型标签 |
| 参考 | 检查了来源，但未检查是否具体到书名/章节/页码 |

影响：即使生成卡片缺了示例卡中的关键字段，质量检查也可能放过。

### 3. 按卡片类型对照

#### 3.1 基础卡：缺少“原文 + 收获 / 评论”的结构

文件：`prompts/note_card.md`

当前格式：

```md
标题：
内容：
参考：
唯一编码：
#基础卡
```

示例卡里的基础卡常见结构：

```md
标题：
原文：
收获 / 评论：
参考：
唯一编码：
#基础卡
```

主要欠缺：

| 欠缺点 | 影响 |
|---|---|
| 没有“原文”字段 | 失去对原始材料的锚定 |
| 没有“收获 / 评论”字段 | 容易变成摘要，而不是阅读卡片 |
| 质量标准强调“不添加主观判断” | 和示例里的“收获”“评论”存在冲突 |
| 定义为“基础信息、背景知识、辅助性内容” | 比示例卡更窄，不能覆盖“读后理解”型基础卡 |

判断：当前基础卡 prompt 更像“背景知识卡”，而示例里的基础卡更像“原文摘录 + 个人理解卡”。

#### 3.2 新知卡：缺少“已知”字段

文件：`prompts/knowledge_card.md`

当前格式：

```md
标题：
新知内容：
应用场景：
与其他知识的联系：
参考：
唯一编码：
#新知卡
```

示例卡中的典型结构：

```md
标题：
已知：
新知：
例子：
参考：
唯一编码：
#新知卡
```

主要欠缺：

| 欠缺点 | 影响 |
|---|---|
| 没有“已知” | 无法明确呈现认知更新前的基线 |
| 没有“例子”作为必填字段 | 新知可能停留在抽象观点 |
| “应用场景”替代了“例子” | 更偏工具化，不一定能检验理解 |
| “与其他知识的联系”是加分项，但不是示例卡必含项 | 可能让卡片变复杂，但核心差异反而不明显 |

判断：新知卡现在强调“反常识”和“应用”，但示例卡更强调“我原来知道什么 → 现在新增什么 → 用例子落地”。

#### 3.3 术语卡：整体较完整，但“术语放在情境中”还可以更强

文件：`prompts/term_card.md`

当前格式：

```md
标题：
定义：
解释：
例子：
参考：
唯一编码：
#术语卡
```

和示例卡高度一致。

不足主要是细节层面：

| 欠缺点 | 影响 |
|---|---|
| “首提出者 / 原始论文”要求较强，但不是所有材料都能提供 | 可能诱发模型补全或幻觉 |
| “情境化”在质量标准中有，但输出字段没有单独承载 | 情境可能被淹没在解释里 |
| 示例里的“标题：什么是执行意图？”更像问题句 | 当前标题要求是术语名称，可能少了提问式可读性 |

判断：术语卡是目前最接近示例卡的 prompt 之一。主要风险是“追溯源头”如果材料没给，模型可能编造。

#### 3.4 金句卡：和示例卡不完全一致，缺少“评论”路径

文件：`prompts/quote_card.md`

当前格式：

```md
标题：
原文：
出处：
仿写：
参考：
唯一编码：
#金句卡
```

示例卡有两种：

```md
标题：
金句：
评论：
参考：
唯一编码：
#金句卡
```

或：

```md
标题：
原句：
仿写：
参考：
唯一编码：
#金句卡
```

主要欠缺：

| 欠缺点 | 影响 |
|---|---|
| 固定要求“仿写” | 适合表达训练，但不适合所有金句 |
| 没有“评论 / 解读”字段 | 对洞见型金句不够友好 |
| 使用“原文”而不是“金句 / 原句” | 可接受，但和示例命名不一致 |
| 目标 5-15 张可能偏多 | 示例卡强调精选，金句卡更应宁缺毋滥 |

判断：当前金句卡 prompt 更偏“金句仿写训练”，示例卡则同时允许评论型金句卡和仿写型金句卡。

#### 3.5 新词卡：完整度很高

文件：`prompts/new_word_card.md`

当前格式：

```md
新异词汇：
原句：
释义：
妙处：
造句：
参考：
唯一编码：
#新词卡
```

示例卡：

```md
新异词汇：
原句：
造句：
参考：
唯一编码：
#新词卡
```

当前 prompt 不仅覆盖了必要元素，还增加了释义和妙处。

判断：新词卡是最完整的。额外字段合理，不会破坏示例卡结构。唯一注意：如果目标是复刻示例风格，`释义` 和 `妙处` 会让卡片更“教学化”；如果目标是提高生成质量，这两个字段是有帮助的。

#### 3.6 事件卡：缺少“行动者 / 反应”，容易摘要化

文件：`prompts/event_card.md`

当前格式：

```md
标题：
时间：
地点：
经过：
意义：
参考：
唯一编码：
#事件卡
```

示例卡单一事件结构：

```md
标题：
时间：
地点：
行动者：
反应：
参考：
唯一编码：
#事件卡-单一事件
```

示例卡多个事件结构：

```md
标题：
日期1：事件
日期2：事件
日期3：事件
参考：
唯一编码：
#事件卡-多个事件
```

主要欠缺：

| 欠缺点 | 影响 |
|---|---|
| 没有“行动者” | 事件中的人物关系不清楚 |
| 没有“反应 / 结果” | 事件缺少收束 |
| 强制“一卡一事件” | 无法很好覆盖示例中的“多个事件时间线” |
| 增加“意义” | 有价值，但可能把事件卡写成评论卡 |
| “用自己的话写” | 对事件卡有风险，可能损失原始叙事细节 |

判断：当前事件卡更像“历史事件摘要卡”；示例事件卡更像“场景还原卡 / 时间线卡”。

#### 3.7 图示卡：缺少真实“图像 / 图表链接”字段

文件：`prompts/graph_card.md`

当前格式：

```md
标题：
图示描述：
各元素说明：
关系说明：
应用场景：
参考：
唯一编码：
#图示卡
```

示例卡：

```md
标题：
说明：
参考：
唯一编码：
#图示卡
![图片]
```

主要欠缺：

| 欠缺点 | 影响 |
|---|---|
| 没有“图像 / 图表链接 / Mermaid / ASCII 图”字段 | 图示卡可能变成“结构说明卡” |
| “图示描述”很强，但不等于图示本身 | 可视化载体不足 |
| 没有要求保留原图或生成可复现图 | 读者无法一眼看到结构 |
| 应用场景不是示例卡必含项 | 可作为增强字段，但不是核心 |

判断：当前图示卡 prompt 强在“文字复现结构”，弱在“图示卡必须有图”。如果最终软件支持图片或 Mermaid，这里是明显缺口。

#### 3.8 人物卡：基本完整，但缺少“贡献 / 研究方向”显式字段

文件：`prompts/person_card.md`

当前格式：

```md
标题：
小传：
代表作品：
参考：
唯一编码：
#人物卡
```

示例卡的实际内容包含：

- 人名
- 小传
- 关键经历
- 研究方向
- 代表作品
- 时间线
- 参考
- 唯一编码

主要欠缺：

| 欠缺点 | 影响 |
|---|---|
| 没有“主要贡献 / 研究方向”独立字段 | 贡献可能埋在小传里 |
| 没有“关键时间线”字段 | 对学术人物发展脉络不够清楚 |
| 代表作品要求 3-5 部 | 适合学者，不一定适合所有人物 |

判断：人物卡可用，但如果目标是“知识源头档案”，建议显式保护 `主要贡献`、`研究方向`、`关键时间线`。

#### 3.9 行动卡：字段比示例更强，但缺少“原理”字段

```md
#行动卡
```

示例行动卡：

```md
标题：
原理：
行动：

唯一编码：
#行动卡
```

主要欠缺：

| 欠缺点 | 影响 |
|---|---|
| 没有“原理” | 行动依据不清楚 |
| 行动步骤很完整 | 这是优点 |
| 增加“预期效果 / 适用场景 / 检验标准” | 很适合产品化，但和示例简洁风格不同 |

判断：行动卡是“执行管理版行动卡”，比示例更工程化。唯一核心缺口是必须补上“原理”，否则行动会脱离知识来源。

#### 3.10 自由卡：完全缺失

示例中有：

- `自由卡-文无第一.md`

但 prompts 目录里没有对应的 `free_card.md`。

自由卡必含元素：

```md
标题：
自由正文：
参考：
唯一编码：
#自由卡
```

影响：系统无法生成“综合性、跨来源、未定型想法”的卡片，只能把这些内容硬塞进基础卡、新知卡或综述卡。

注意：`all_cards.md` 有“综述卡”，但“综述卡”不等于“自由卡”。

| 类型 | 核心 |
|---|---|
| 自由卡 | 开放表达、跨材料融合、未定型想法 |
| 综述卡 | 结构化总结、跨章节整合 |

#### 3.11 索引卡：是扩展卡型，不在示例卡必要元素中

文件：`prompts/index_card.md`

这个 prompt 本身结构清晰：

```md
标题：
人物索引：
术语索引：
概念索引：
著作索引：
事件索引：
参考：
唯一编码：
#索引卡
```

和示例卡体系相比：

- 它不是示例目录里的核心卡型；
- 但作为软件功能是合理扩展；
- 需要在 `quality_check.md` 里加入检查规则，否则质量无法闭环。

### 6. 最值得优先补强的方向

如果不改 prompt，只从诊断角度看，最应该关注这 6 个方向：

1. 所有卡统一保留：标题、参考、唯一编码、类型标签。
2. 新知卡必须体现：已知 → 新知 → 例子。
3. 基础卡必须体现：原文 / 内容 → 收获 / 评论。
4. 事件卡必须体现：时间、地点、行动者、行动、反应。
5. 图示卡必须有某种可视化载体：图片链接、Mermaid、ASCII 图或原图占位。
6. 质量检查必须覆盖所有卡型，而不是只覆盖部分卡型。

一句话判断：当前 prompt 已经具备“生成知识卡”的能力，但和示例卡相比，仍偏向摘要化、解释化、产品化字段；示例卡更强调原始材料锚定、个人认知变化、场景还原、表达训练和唯一归档。
